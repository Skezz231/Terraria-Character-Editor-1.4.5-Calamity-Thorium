# Security Policy

## Supported versions

Security fixes are generally applied to the latest active version of the project.

## Reporting a vulnerability

If you believe you found a vulnerability, please report it privately.

Include:
- a clear description of the issue
- affected files or functionality
- reproduction steps if available
- potential impact
- screenshots or proof of concept if helpful

Please do not post security-sensitive details in a public issue.

## Response goals

When a valid report is received, the goal is to:
- confirm receipt
- assess severity
- prepare a fix if needed
- publish changes responsibly

## Scope examples

Relevant reports may include:
- unsafe file handling
- path traversal issues
- insecure loading behavior
- untrusted input processing problems
- accidental exposure of sensitive local data
